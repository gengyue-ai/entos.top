# AI 采纳顾问——能力体系（详细版）

## 概述

做这件事，不需要你成为"AI 专家"，但需要你在四个维度上各有一定深度。

这四层不是平级的，是有优先级的：

```
行业判断力 > 流程拆解力 > 技术选型力 > 落地执行力
```

下面逐层拆解，精确到"能不能"、"会不会"、"有没有"。

---

## 第一层：行业判断力（最重要）

**定义：** 能看出一个行业的某个业务环节，值不值得用 AI 重做。

### 具体能力

| 能力项 | 具体表现 | 判断标准 |
|--------|----------|----------|
| 识别重复性 | 看到一个流程，能快速判断"这是人脑在重复劳动，还是需要真正判断力" | 能说出具体哪几步是人肉做的 |
| 估算规模 | 能估算这个流程在整个行业里一年要花多少人工 | 能说出"这个环节全行业每年 X 亿小时" |
| 判断痛点 | 能区分"这个老板觉得痛"和"这个痛点值得用 AI 解决" | 能说清楚为什么有些痛点 AI 解决不了 |
| 识别决策链 | 知道一个项目的关键决策者是谁、TA 关心什么 | 能画出采购决策链上的角色和诉求 |

### 你现在的水平

| 能力项 | 你 | 说明 |
|--------|:--:|------|
| 识别重复性 | ✅ 强 | 20 年经验，流程对不对一看就懂 |
| 估算规模 | ✅ 强 | 做过项目，知道生意怎么算 |
| 判断痛点 | ✅ 强 | 政府项目经验让你对"真痛"很敏感 |
| 识别决策链 | ⚠️ 需要练习 | 技术人出身，对"谁说了算"的判断需要刻意练 |

## 第二层：流程拆解力（基本功）

**定义：** 把一个业务场景从"人是怎么干的"拆成"哪几步可以用 AI 干"。

### 具体能力

| 能力项 | 具体表现 | 能做到什么程度才算及格 |
|--------|----------|------------------------|
| 画流程图 | 能用一个工具画出重构前/后的流程图 | 不是标准的 UML，能让客户看懂就行 |
| 标数据依赖 | 能标出每一步需要什么数据、数据从哪来 | 能说清楚现有系统里有啥数据、缺啥 |
| 算时间成本 | 能估算每一步的人工耗时和出错率 | 能给出"现在这步每周花 3 人天，错误率 15%" |
| 设计新流程 | 能设计出"AI 负责哪几步、人负责哪几步" | 能让客户觉得"这样干确实比现在好" |

### 练习方法

拿你自己熟悉的业务场景，画一个"重构前"和"重构后"的流程图。比如：

- 以前：一个商场怎么处理商户的租金结算
- 现在：用 AI 怎么做

画完拍照发给我，我帮你修。

### 你现在的水平

| 能力项 | 你 | 说明 |
|--------|:--:|------|
| 画流程图 | ✅ 会 | 架构图你画过了 |
| 标数据依赖 | ✅ 会 | 做过系统集成，知道数据从哪来 |
| 算时间成本 | ✅ 会 | 项目经验 |
| 设计新流程 | ✅ 会 | HelloCut 本身就是个重构 |

## 第三层：技术选型力（够用就行）

**定义：** 知道 AI 能做什么、不能做什么，能帮客户选对工具。

### 具体能力

| 能力项 | 具体表现 | 及格标准 |
|--------|----------|----------|
| 理解能力边界 | 知道 AI 擅长什么（分类、生成、提取、总结）和不擅长什么（精确计算、因果推理） | 不会向客户承诺 AI 做不到的事 |
| 认识工具 | 知道主流 AI 工具各自适合什么场景（大模型、MCP、RAG） | 能给客户推荐 1-2 个具体方案 |
| 规划技术方案 | 能为一个场景设计出"够用"的技术路线，不上大炮打蚊子 | 方案里没有"我们用大模型解决一切"这种话 |
| 判断复杂度 | 能预估一个 AI 流程的落地难度（数据好不好、系统能不能接、人愿不愿意用） | 能跟客户说清"这个场景做起来要多久、难在哪" |

### 你的现状

| 能力项 | 你 | 说明 |
|--------|:--:|------|
| 理解能力边界 | ✅ 强 | 你在用 Hermes 和 MCP，比大部分技术人更理解边界在哪 |
| 认识工具 | ✅ 够用 | Hermes + MCP + FireCrawl 已经是一个完整的技术栈 |
| 规划技术方案 | ✅ 够用 | 你有系统架构经验 |
| 判断复杂度 | ⚠️ 需要练 | 真正落地的坑只能靠做才知道 |

### 你不需要的能力

- 不需要会写大模型
- 不需要会写 prompt
- 不需要懂训练和微调

## 第四层：落地执行力（关键差距）

**定义：** 能把方案从纸面变成可运行的东西。

### 具体能力

| 能力项 | 具体表现 | 及格标准 |
|--------|----------|----------|
| 快速原型 | 能在 1-2 天里跑出一个"虽然糙但能用"的东西 | 客户看到后说"这东西直接能用" |
| 接数据 | 能把客户的现有系统数据接入 AI 流程 | 能处理 CSV、API、数据库、人工导出 |
| 与业务方沟通 | 能用业务语言跟客户沟通，不炫技 | 客户听不懂的话不说 |
| 推动采纳 | 能推动客户团队真正用起来，而不是"买完放着" | 知道谁会用、谁会抵制、怎么解决 |

### 你的现状

| 能力项 | 你 | 说明 |
|--------|:--:|------|
| 快速原型 | ✅ 会 | 你能在几天里搭出一个能用的东西 |
| 接数据 | ✅ 会 | 你做过系统集成 |
| 与业务方沟通 | ✅ 强 | 这是你的核心优势 |
| 推动采纳 | ⚠️ 需要练 | 这跟纯技术交付不同，需要刻意做 |
