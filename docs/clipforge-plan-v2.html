ClipForge · 复盘与完整方案

2026-07-12 · 版本 1.0

一、复盘

1.1 发生了什么

从开始到今天，经历了以下阶段：

阶段时长结果
Blender API 探索～4h失败 Blender 5.1.2 FFMPEG 编译不完整
隧道+页面架构～2h部分可用 页面和隧道可运行
本地 Agent 开发～2h未完成 依赖未就绪
FFmpeg 混剪测试～3h可行 基础拼接已出片

1.2 核心问题

1. 跳步。 没先确认核心链路能不能跑（Blender CLI 出片），直接写页面、隧道、Agent 三层代码。每层都有 bug，叠在一起没法排查。
2. 猜测代替验证。 Blender 5.1.2 的 API 一直靠猜，直到最后才发现它有 FFMPEG 编译问题。
3. 让你反复做体力活。 删文件→重下载→重运行，你成了调试工具。
4. 脱离实际需求。 想要的是"政府项目品质宣传片"，我在纠结"Blender 能不能输出 MP4"。

1.3 关键发现

Blender 5.1.2： FFMPEG 输出未完全编译，无法通过 Python API 设视频格式。CLI -F FFMPEG 能出视频但默认 MKV 容器。
FFmpeg： 版本 8.1.2，完整支持 H.264、AAC、xfade 转场、drawtext 字幕、音视频混流。可作为主力渲染引擎。
架构： SaaS 页面 + 云端调度 + 本地 Agent + 本地渲染（FFmpeg/Blender）是正确方向。

四、前后端方案

4.1 SaaS 页面（已有原型，需完善）

entos.top/clipforge/ 深色主题，包含：

  • 仪表盘 — Agent 连接状态、任务列表
  • 配置面板 — 上述全部参数的输入控件（下拉框、滑块、文本输入）
  • 模板选择 — 一键切换预设
  • 进度条 — 实时渲染进度
  • 日志 — FFmpeg 输出实时显示
  • 历史记录 — 已完成的任务列表

4.2 云端调度（已有，需完善）

FastAPI + SQLite，路由：

API方法说明
/api/agent/statusGETAgent 在线状态
/api/agent/taskPOST创建任务（含全部参数）
/api/agent/progress/{id}GET任务进度
/api/agent/historyGET历史任务
/api/agent/cancel/{id}POST取消任务

4.3 隧道（已有）

WebSocket 隧道（阿里云:8768 ↔ 本地 Agent），任务通过数据库轮询转发。

4.4 本地 Agent（需重写）

功能实现优先级
接收任务WebSocket 接收任务参数P0
素材分析FFmpeg 场景检测 + 运动分析 + 音量分析P1
智能编排按精彩度排序 + 节奏控制P1
